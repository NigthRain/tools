最佳提示词实践 (Best Practice Prompt)
要生成我们最终版本的 Typora 主题，一个好的提示词需要包含以下几个关键部分：
角色定义 (Persona)：明确要求 AI 扮演一个具备特定知识和技能的角色。
核心理念 (Core Philosophy)：设定主题的整体风格和高级目标。
技术规范 (Technical Specs)：给出具体的、不可妥协的技术实现要求（如使用变量、@font-face）。
详细样式 (Detailed Styles)：分模块、逐一说明每个元素的精确视觉要求（字体、颜色、间距等）。
输出格式 (Output Format)：规定最终交付物的形式。
以下是我为您构建的最佳实践提示词：
提示词 (Prompt)
## 1. 角色与任务定义 (Role and Task Definition)
你是一位精通前端开发和 CSS 设计的专家，尤其擅长为 Typora 编写自定义主题。你的任务是根据以下详细规范，为我生成一个名为 "Screen LapisW Theme" 的、完整且高质量的 Typora 自定义 CSS 文件。
## 2. 核心设计理念 (Core Design Philosophy)
风格: 专业、简洁、学术，适合长时间的沉浸式阅读与写作。
主色调: 以一种专业且不失活力的青金石蓝色 (#4870ac) 作为贯穿始终的主题色。
背景: 采用纯白背景 (#ffffff)，确保内容区域有最高的对比度和可读性。
排版: 现代网页排版风格，段落之间通过垂直间距分隔，首行不缩进。
## 3. 详细技术与样式规范 (Detailed Technical and Style Specifications)
3.1. 整体结构与最佳实践
CSS 变量: 必须在 :root 中定义所有核心颜色，以便于维护和修改。
代码结构: 将整个 CSS 文件划分为逻辑清晰的、带编号的模块（如：字体定义、基础样式、段落与标题、代码块等），并为每个模块添加详尽的中文注释。
3.2. 字体系统 (非常重要)
加载策略: 必须采用“本地字体优先，网络字体备选”的策略，以实现最佳性能。
实现方式: 必须使用 @font-face 规则，通过 src: local('...') 检查本地字体，url('...') 作为备选。禁止使用 @import，以避免阻塞渲染。
具体字体栈:
段落 (默认): font-family: "Times New Roman", "文津宋体 第0平面", "文津宋体", "SYPX-ZhenSong", serif;
SYPX-ZhenSong 的网络字体 URL: https://cdn.jsdelivr.net/gh/Yin-s/typora-theme-lapis@main/lapis/font/SYPX-ZhenSong.woff2
H1 标题: font-family: "抖音美好体", "DouyinSansBold", sans-serif;
DouyinSansBold 的网络字体 URL: https://chinese-font.netlify.app/fonts/dymh/DouyinSansBold.woff2
H2 标题: font-family: "等距更纱黑体 Slab SC", "MiSans", -apple-system, BlinkMacSystemFont, "Segoe UI", "Microsoft YaHei", "PingFang SC", sans-serif; (无需网络字体)
H3 标题: font-family: "霞鹜文楷 GB 屏幕阅读版", "霞鹜文楷 屏幕阅读版", "LXGWWenKaiMonoScreen", sans-serif;
LXGWWenKaiMonoScreen 的网络字体 URL: https://cdn.jsdelivr.net/gh/Yin-s/typora-theme-lapis@main/lapis/font/LXGWWenKaiMonoScreen-Regular.woff2
H4 标题: font-family: "文津宋体", "SYPX-ZhenSong", serif; (使用已定义的思源宋体备选)
H5 标题: 与 H2 字体完全相同。
H6 标题: 与段落字体完全相同。
表格文字: 与 H3 字体完全相同。
代码块与行内代码: font-family: "Sarasa Term Slab SC", "LXGWWenKaiMonoScreen", monospace; (使用已定义的霞鹜文楷备选)
3.3. 颜色方案
--text-color: #40464f (正文)
--primary-color: #4870ac (主题蓝)
--block-bg-color: #EDF1F7 (代码块、引用块背景)
H1 标题颜色: #4870AC
H3 标题颜色: #0D3869
H4 标题颜色: #002FA7
3.4. 布局与间距
内容区最大宽度 max-width: 1200px;。
全局行高 line-height: 1.7em;。
段落 p 的下外边距 margin-bottom: 0.3em;，无首行缩进。
3.5. 标题 (H1-H6)
H1: 居中对齐。
H2: display: inline-block;，背景色为主题蓝，文字为白色。
H3: 加粗。
H4: 加粗。
H5: 居中对齐，font-weight: normal;。
3.6. 代码块 (Code Fences)
交互: 鼠标悬停时，有轻微上浮和阴影效果。
装饰: 块顶部有模拟 macOS 的三个窗口控制点（使用内联 SVG 实现）。
语言标签 (最关键):
必须在右上角显示代码语言。
必须使用 z-index: 10;，确保它显示在所有代码文本的最上层。
样式为浅色背景 (rgba(255, 255, 255, 0.85))，主题蓝色文字，带轻微圆角和内边距。
定位为 top: .5rem; right: .7rem;。
## 4. 输出要求 (Output Requirements)
请根据以上所有规范，生成一份单一、完整的 CSS 文件。在文件的开头，请添加一个注释块，说明主题名称、版本号 (2.2.1) 和核心特性摘要。
